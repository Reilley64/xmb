# Spec: Settings

## Overview

The Settings column (always first in the XMB icon row) provides two actions: selecting the ROM base directory and refreshing the ES-DE system config. No other columns are blocked while these are unconfigured — they simply show empty lists.

## Settings Items

| Item | Action |
|------|--------|
| Choose ROMs folder | Triggers permission check + folder picker |
| Refresh system config | Re-fetches ES-DE XML and rescans ROMs |

## ROM Folder Selection Flow

1. User confirms "Choose ROMs folder"
2. `FolderPicker.checkManageStoragePermission()` is called
   - Returns `true` if Android API < 30, or if `MANAGE_EXTERNAL_STORAGE` is already granted
3. If not granted:
   - `FolderPicker.requestManageStoragePermission()` opens the system settings screen (`ACTION_MANAGE_APP_ALL_FILES_ACCESS_PERMISSION`)
   - Flow ends here; user must return to the app and try again after granting
4. If granted:
   - `@react-native-documents/picker` fires `ACTION_OPEN_DOCUMENT_TREE`
   - User picks a directory
   - The content URI (`content://...`) is stored in AsyncStorage under `romSafTreeUri`
   - `contentUriToPath()` converts the URI to a plain file path (possible because `MANAGE_EXTERNAL_STORAGE` is granted — bypasses scoped storage)
   - File path stored under `romBasePath`
   - `useRomLibrary.setRomBasePath(path)` triggers an immediate ROM scan

## System Config Refresh Flow

1. User confirms "Refresh system config"
2. `systemConfig.fetchSystemConfig({ forceRefresh: true })` fetches fresh ES-DE system XML (bypasses cache)
3. `findRules.fetchFindRules({ forceRefresh: true })` fetches fresh find rules XML (bypasses cache)
4. Both are re-cached in AsyncStorage
5. `useRomLibrary.refresh()` reloads the game library UI

## AsyncStorage Keys

| Key | Value |
|-----|-------|
| `romBasePath` | Absolute file path, e.g. `/storage/emulated/0/ROMs` |
| `romSafTreeUri` | Content URI, e.g. `content://com.android.externalstorage.documents/tree/primary%3AROMs` |
| `esSystemConfig_v2` | JSON array of `EsSystem[]` |
| `esFindRules_v1` | JSON object of `FindRules` |

## Native Module: FolderPickerModule

Generated by `plugins/withNativeModules.js`.

| Method | Returns | Description |
|--------|---------|-------------|
| `checkManageStoragePermission()` | `boolean` | True if storage permission granted or API < 30 |
| `requestManageStoragePermission()` | `void` | Opens system settings for permission grant |

The actual directory picker (`ACTION_OPEN_DOCUMENT_TREE`) is handled by `@react-native-documents/picker`, not this native module.

## Android Permissions

Added to `AndroidManifest.xml` by `plugins/withNativeModules.js`:

| Permission | Purpose |
|------------|---------|
| `MANAGE_EXTERNAL_STORAGE` | Full filesystem access on Android 11+ |
| `SYSTEM_ALERT_WINDOW` | Reserved for future use |

## UI Components

| Component | File |
|-----------|------|

Settings items use the default `assets/settings.webp` icon.

## Edge Cases

- **No ROM path set**: Columns for all systems exist but show empty game lists. The app is fully navigable.
- **Permission denied**: The flow stops after opening system settings. No error is surfaced beyond the navigation away — user retries by selecting the item again.
- **Refresh while scanning**: Refresh uses `Effect.runPromise` so it runs sequentially after the initial scan completes.
