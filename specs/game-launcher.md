# Spec: Game Launcher

## Overview

When a user confirms a game, the app resolves the system's ES-DE launch commands into concrete Android intents, then fires them. If multiple emulator options exist, the user picks one via the EmulatorPicker overlay. Play stats are written to SQLite on success.

## LaunchIntent Type

```ts
interface LaunchIntent {
  label: string
  candidates: { pkg: string; activity: string }[]
  action: string | null
  dataUri: string | null
  extras: Record<string, string>
  clearTask: boolean
  clearTop: boolean
}
```

## Launch Flow

```
User confirms game
  → resolveCommands(system.commands, game.file_path)
      → parseEsCommand() × N          // one per command variant
  → intents.length === 1
      ? launchRom(game, intents[0])   // fire immediately
      : show EmulatorPicker           // user picks intent
          → launchRom(game, intent)
```

## Command Resolution (`resolveCommands`)

File: `src/services/launcher.ts`

1. Loads find rules from cache (or fetches if missing)
2. Loads SAF tree URI from AsyncStorage (`romSafTreeUri`)
3. For each `{ label, command }` in `system.commands`, calls `parseEsCommand()`
4. Returns `LaunchIntent[]` (one per command variant)

## Command Parsing (`parseEsCommand`)

Parses ES-DE command format: `%EMULATOR_NAME% [flags] [token=value...]`

### Emulator name extraction

`%EMULATOR_NAME%` (where `NAME` is the uppercase emulator key) is the first token. The emulator name is looked up in `FindRules` to get `candidates[]`.

### Supported tokens

| Token | Effect |
|-------|--------|
| `%ACTIVITY_CLEAR_TASK%` | Set `clearTask = true` |
| `%ACTIVITY_CLEAR_TOP%` | Set `clearTop = true` |
| `%ACTION%=value` | Set intent `action` |
| `%DATA%=value` | Set intent `dataUri` |
| `%EXTRA_KEY%=value` | Add `{ KEY: resolvedValue }` to extras |

### Token value resolution

| Placeholder | Resolved value |
|-------------|---------------|
| `%ANDROIDPACKAGE%` | First candidate's package name |
| `%ROM%` | Absolute file path |
| `%ROMSAF%` | SAF document URI (or `file://` fallback) |
| `%ROMPROVIDER%` | `file://` URI |

## SAF Document URI (`buildSafDocUri`)

Converts an absolute file path to an Android SAF document URI using the stored tree URI:

1. Strip `/storage/emulated/0/` prefix → replace with `primary:`
2. Append to `DocumentsContract.buildDocumentUriUsingTree()` form

## Intent Firing (`launchRom`)

1. Acquires a semaphore (serializes concurrent launch attempts)
2. Iterates `intent.candidates` in order, attempting each:
   - Package name → `packageName`
   - Activity → `className`
   - Flags: `FLAG_GRANT_READ_URI_PERMISSION` always; `FLAG_ACTIVITY_CLEAR_TASK` / `FLAG_ACTIVITY_CLEAR_TOP` if set
   - Calls native `GameLauncherModule.launchIntent(params)` (see native modules spec)
3. First successful candidate wins; remaining are skipped
4. If all candidates fail → returns `AllCandidatesFailedError`
5. On success: increments `play_count` and writes `last_played = Date.now()` to SQLite

## Native `GameLauncherModule`

Kotlin module generated by `plugins/withNativeModules.js`.

Method: `launchIntent(params: ReadableMap)`

| Param | Type | Required | Description |
|-------|------|----------|-------------|
| `pkg` | string | yes | Android package name |
| `activity` | string | yes | Fully-qualified activity class |
| `safTreeUri` | string | no | SAF tree URI for persistent URI permissions |
| `action` | string | no | Intent action |
| `dataUri` | string | no | Intent data URI |
| `clearTask` | boolean | no | `FLAG_ACTIVITY_CLEAR_TASK` |
| `clearTop` | boolean | no | `FLAG_ACTIVITY_CLEAR_TOP` |
| `extras` | map | no | Key/value extras passed to the emulator |

If `safTreeUri` is provided, the module takes persistent read URI permission before launching.

## EmulatorPicker UI

Component: `src/components/organisms/emulator-picker.tsx`

- Shown when `pendingLaunch` has more than one intent
- Slide-in from right, occupies ~32% of screen width
- Left gradient dims the game list (transparent → 55% black)
- Up/Down to move between options, Confirm to select, Back to cancel
- On cancel: `pendingLaunch` is cleared with no launch

## Error Types

| Error | Trigger |
|-------|---------|
| `AllCandidatesFailedError` | All `pkg/activity` pairs rejected by Android |
| `LaunchError` | Single intent attempt failed |
| `ConfigUnavailableError` | Find rules could not be loaded |

## Play Stats

On successful launch, the following SQL runs:

```sql
UPDATE games
SET play_count = play_count + 1,
    last_played = :now
WHERE id = :id
```

`last_played` is a Unix timestamp in milliseconds. This is what powers the Recent column in the XMB.
